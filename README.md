# 溢价率日常看板

这是一个面向 LOF / QDII 场内溢价率观察的本地网站原型，目标是替代直接拿 T-2 净值估溢价率的粗糙做法。

当前版本已支持 20 只基金的日常看板，并为每只基金维护独立的误差记录和独立修正模型。

## 当前能力

- 启动时自动抓取最新基金净值、历史净值和场内价格
- 日度资料缓存到 .cache/fund-sync/daily，避免一天内重复抓取
- 日内场内价格、USD/CNY 和 161128 前十大持仓报价缓存到 .cache/fund-sync/intraday
- 自动记录昨日估值与后续真实净值的差值，并累积误差历史
- 首页改为表格主看板，展示现价、估值、净值日期、现价时间和限购状态
- 详情页展示每只基金自己的误差折线图和模型状态
- 每只基金维护独立模型，不与其他基金共用参数
- 161128 额外保留持仓级估值实验室，并自动带入 USD/CNY 与美股前十大持仓报价

## 方法简述

首页自动模型当前使用：

自动估值 = 最近官方净值 × (1 + alpha + betaLead × 场内当日涨跌幅)

其中：

- 场内当日涨跌幅 = 当前场内价格 / 昨收价 - 1
- alpha、betaLead 由各基金自己的误差历史单独更新
- 当真实净值发布后，系统会把当日预测误差回写到该基金自己的模型

161128 详情页额外提供持仓模式：

- 以官方 T-1 净值为锚点
- 使用前十大持仓权重和代理篮子做净值贡献拆解
- 叠加汇率变化、费用拖累、人工修正和独立细模型修正

## 后续建议

1. 给每只基金补更贴近标的资产的代理因子，而不是只用场内领先因子。
2. 将 161128 的持仓自动抓数接进来，减少详情页手动输入。
3. 给误差历史增加导出、回测窗口和分段统计。
4. 后续可以升级成递归最小二乘、卡尔曼滤波或树模型，而不是当前的轻量在线线性模型。

## 本地运行

现在最简单的主入口是直接双击 start-site.cmd。

它会完成这几件事：

- 启动前先同步一次数据
- 启动本地网站
- 自动打开浏览器
- 每 60 秒后台刷新一次同步数据

如果你要手动在终端里运行，也可以继续用：

```powershell
npm install
npm run dev
```

默认开发地址：

```text
http://localhost:5173
```

如果 5173 被占用，Vite 会自动顺延到 5174、5175 等端口。

如果只想手动刷新同步数据，可以执行：

```powershell
npm run sync:data
```

如果你想直接运行自动刷新版本，也可以执行：

```powershell
npm run app:start
```

## 免费分享方案

最省事的分享方式是 GitHub Pages：

- 网站本体按静态站点发布，不需要别人访问时你的电脑一直开着
- 数据同步和构建由 GitHub Actions 定时执行
- 别人直接打开网址即可查看最近一次更新后的页面
- 静态站点没有数据库和登录入口，攻击面比自己开一个公网服务小很多

### 推荐做法

1. 把这个项目推到 GitHub 仓库。
2. 在仓库设置里启用 GitHub Pages，并选择 GitHub Actions 作为来源。
3. 保留仓库中的工作流文件 .github/workflows/deploy-pages.yml。
4. 后续只要推代码，或者等定时任务触发，Pages 就会自动更新。

### 最短上手步骤

如果你还没怎么用过 GitHub，可以直接按这个顺序做：

1. 去 GitHub 新建一个空仓库，例如 premium-estimator-site。
2. 在本地打开当前项目目录。
3. 依次执行下面这些命令，把项目推上去。

```powershell
git init
git add .
git commit -m "init premium estimator site"
git branch -M main
git remote add origin https://github.com/<你的用户名>/premium-estimator-site.git
git push -u origin main
```

4. 推送完成后，打开 GitHub 仓库页面。
5. 进入 Settings -> Pages。
6. Source 选择 GitHub Actions。
7. 等待 Actions 里的 Deploy Pages 工作流第一次跑完。

跑完后你就会拿到公开网址：

```text
https://<你的用户名>.github.io/premium-estimator-site/
```

如果你之前电脑里没有安装 Git，可以先装 Git for Windows：

```text
https://git-scm.com/download/win
```

### 更新频率

当前工作流会在工作日 UTC 01:00-07:59 每 15 分钟更新一次。

换算成北京时间，大致覆盖：

- 09:00 - 15:59

这意味着你早上没开电脑，别人也能看到 GitHub 上最近自动更新后的数据。

### 网址形式

因为这是静态站点，路由使用哈希模式，分享出去的地址会是这种形式：

```text
https://<你的用户名>.github.io/<仓库名>/
```

点进基金详情后会变成：

```text
https://<你的用户名>.github.io/<仓库名>/#/fund/161128
```

这样可以避免 GitHub Pages 对前端路由的 404 问题。
